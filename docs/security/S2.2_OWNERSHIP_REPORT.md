# BIBI Cars — S2.2 STAFF OWNERSHIP / IDOR SCOPING REPORT

> PHASE SECURITY · Wave S2 · Stage 2.2
> Goal: within the authenticated **staff** tier, constrain object access so a
> manager sees only their own clients, admin/master/owner see all, and customers
> never see others' objects. Companion to `IDOR_REPORT.md`.

---

## 1. Role policy (enforced)

| Role | Customer-scoped objects |
|---|---|
| owner / master_admin / admin | **all** |
| team_lead | **all** (true team-scoping = residual, see §4) |
| manager | **only** customers where `managerId == manager.id` |
| customer | only own objects (gate + cabinet ownership) |
| anonymous | none (gate → 401) |

Canonical helper: **`app/services/staff_acl.py`**
- `staff_can_see_customer(user, customer) -> bool`
- `customer_scope_filter(user) -> mongo_filter`
Unit-tested (owner/admin/team_lead → True; manager-owns → True; manager-other → False; unassigned → False; customer → False).

Ownership/role scoping helpers are already wired into **25 call sites** across:
- `server.py` — leads (`_lead_role_scope`, `_can_user_see_lead`, `_can_user_touch_lead`), customers (`_customer_scope_filter`, `_can_user_see_customer`)

Combined with the **default-deny gate (S2.1)**, every object route is now at minimum **staff-only**, and the main customer/lead/deposit/file surfaces additionally enforce manager ownership.

---

## 4. Residual (tracked, lower risk)

1. **team_lead true team-scoping** — currently team_lead = "see all" because **no team-membership model exists** (no `teamId` / `reportsTo` linking managers to a team_lead). Implementing "team_lead sees only their team" requires adding that model first. *Recommend: confirm desired team structure, add `teamId` to staff + customers, then switch team_lead from all-access to `teamId`-scoped in `staff_acl.py` (single chokepoint).*
2. **Long-tail object routes** — remaining `{id}` object routes (e.g. `/api/deals/{id}`, `/api/documents/{id}`, `/api/history/report/{id}`) are now staff-gated; per-route manager ownership should be applied incrementally using `staff_can_see_customer` / `customer_scope_filter`. None are unauthenticated (gate closed that).
3. Apply `customer_scope_filter` to any staff *list* endpoint that returns customer rows without it (audit on next pass).
