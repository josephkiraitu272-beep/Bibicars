# PHASE SECURITY — Wave S3.1 — Upload Security (COMPLETE)

**Status:** ✅ DONE · **Scope:** server-authoritative validation of every file
upload. The client-supplied `Content-Type` is no longer trusted anywhere.

## Threat model closed
- **MIME spoofing** — `image.jpg` that is really HTML/PHP/EXE.
- **Stored XSS / RCE** — `payload.png` containing `<script>` / `<?php` / `<svg>`.
- **Double / dangerous extensions** — `img.jpg.php`, `doc.pdf.js`, `a.png.html`.
- **Executable / archive smuggling** — ELF, PE (`MZ`), Mach-O, ZIP/RAR/7z, gzip.
- **Path traversal** in filenames — `../../etc/passwd`, control chars, header injection.
- **Oversize uploads** — per-category ceilings.

## Defense (order of checks)
1. Non-empty + hard size ceiling.
2. Filename sanitisation (strip dirs, `..`, control/header-unsafe chars, len≤200).
3. Dangerous/double-extension denylist (exe, php, js, html, svg, zip, …).
4. Extension allowlist (final ext must be permitted).
5. Content-signature scan — reject HTML/SVG/script (`<script`, `<?php`, `<svg`, `javascript:`) and executable/archive magic.
6. **Magic-byte sniffing** (`filetype`) → the REAL type decides the stored mime/ext.
7. Per-category size limit.

## Mandated limits (BIBI Cars)
| Category | Limit |
|----------|-------|
| Photos (jpg/png/webp/gif) | **10 MB** |
| PDF | **25 MB** |
| Other documents (office/text) | **25 MB** |
| Hard ceiling (`MAX_ANY`) | 25 MB |

SVG is **forbidden** everywhere (raster images only).

## Verification
- **Unit:** 16 edge cases on `validate_image_upload` (valid jpg/png/gif/webp;
  html/php/svg/elf/zip disguised as image; double extensions; empty; oversize;
  hero gif rejection) — all PASS.
- **Live smoke (admin token, strict mode):** all 4 content endpoints accept a
  real PNG and reject HTML-as-PNG, PHP-as-JPG, SVG-as-PNG; hero correctly
  rejects GIF with `unsupported image type: image/gif`.

## Files changed
- `app/services/upload_security.py` — added `validate_image_upload()`; limits set to 10/25/25 MB.
- `app/routers/content.py` — 4 endpoints now use magic-byte validation (was trusting `image.content_type`).
