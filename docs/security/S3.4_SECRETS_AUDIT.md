## 2. Safe fixes applied (this wave)

1. **Resend key de-hardcoded** — `notifications.py`
   - `RESEND_API_KEY_FALLBACK = "re_…"` → `os.environ.get("RESEND_API_KEY_FALLBACK","")`.
   - Verified live email still resolves the key from **DB** (`integration_configs.resend`,
     `source: db`) → **no regression** (send path consults DB first).
2. **Stripe `webhookSecrets[]` masking** — `app/routers/admin_integrations.py`
   - Added `webhookSecrets` to the stripe secret-field set.
   - Masking loop now masks **list-typed** secrets element-wise.
   - PATCH preservation now keeps stored list when every element is a masked
     placeholder (`…suffix`) → round-trip save no longer wipes the secret.
   - Re-tested live: `webhookSecrets: ['…i8R9kywX']` (no full `whsec_` in response).
3. **Git hygiene** — `git rm --cached backend/.jwt_secret memory/stripe_webhook.txt`
   + `.gitignore` entries (`backend/.jwt_secret`, `*.jwt_secret`,
   `memory/stripe_webhook.txt`). Files remain on disk → running app unaffected.
4. **Staff passwords de-hardcoded** — `server.py` `DEFAULTS` password literals →
   `""`; values now read from env (`BIBI_ADMIN_PASSWORD` / `BIBI_MANAGER_PASSWORD`
   / `BIBI_TEAM_LEAD_PASSWORD`) set in gitignored `.env` (current values kept →
   logins unchanged; verified admin/manager/team-lead+OTP/customer). A startup
   warning now fires if a role has no password from env.
5. **Demo customer seed gated** — wrapped in `SEED_DEMO` (auto-OFF when
   `ENVIRONMENT=production`); ON for preview QA.

> No live Stripe/Resend/JWT/OAuth credential value was changed or rotated. Staff
> passwords were **relocated** (not rotated) — same values, now only in env.
