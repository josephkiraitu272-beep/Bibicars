# PHASE SECURITY — Wave S3.2 — Rate Limiting (COMPLETE)

**Status:** ✅ DONE · **Engine:** `slowapi`/`limits` (`MovingWindowRateLimiter`
+ in-memory `MemoryStorage`, per uvicorn worker). **No Redis** (approved — avoids
extra infra/failure point at current scale).

## Architecture
- New module `app/middleware/rate_limit.py` + one HTTP middleware in `server.py`,
  registered **after** the access gate → it is the **outermost** layer and runs
  **first** (abusive traffic throttled before any auth / business logic).
- **Client key** (approved order): `X-Forwarded-For` (left-most) → `X-Real-IP`
  → socket peer. Correct behind the Kubernetes Ingress.
- **On breach:** `HTTP 429` + JSON `{"detail":"Rate limit exceeded","retry_after":N}`
  and headers `Retry-After`, `X-RateLimit-Limit`, `X-RateLimit-Remaining`,
  `X-RateLimit-Reset`. Successful responses also carry the `X-RateLimit-*` headers.
- **Whitelist (never limited):** `/api/ext/*`, `/api/vesselfinder/*` (extension
  HMAC tier + workers) and **every route not explicitly listed** (opt-in design).

## Verification (live smoke — preview, strict mode)
- **`POST /api/auth/login`** — 5 × `401` then `429` (`retry_after: 59`); 429 carries
  `Retry-After` + `X-RateLimit-*` + JSON body. ✅
- **`GET /api/vin/search`** — `200` with `X-RateLimit-Limit: 30, Remaining: 29`. ✅
- **`POST /api/ext/heartbeat`** — `401` (HMAC), **never 429** after 12 hits → whitelist intact. ✅
- **`GET /api/vehicles`** — not throttled after 8 hits → unlimited routes untouched. ✅
- **Legitimate admin login** — `200`, `X-RateLimit-Remaining: 4` (not blocked). ✅
- **Unit:** 30 path/method classification cases + 3/15min enforcement (4th hit blocked) — all PASS.

## Notes / caveats
- In-memory store is per-worker and resets on restart — acceptable for brute-force
  defence at current scale; revisit Redis only if multi-worker horizontal scaling is added.
- Clients behind shared NAT share a key (accepted tradeoff vs. correctness behind Ingress).
