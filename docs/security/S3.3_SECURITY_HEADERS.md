# PHASE SECURITY — Wave S3.3 — Security Headers + CSP (Report-Only) (COMPLETE)

**Status:** ✅ DONE · Headers ENFORCED globally; CSP shipped **Report-Only**
(non-breaking) with a working violation sink. HSTS deferred to prod cut-over.

## What shipped
### 1. Global safe headers (ENFORCED — every response)
Backend middleware `app/middleware/security_headers.py` + `_security_headers`
http-middleware in `server.py` (outermost), and the same set on the SPA via
`craco.config.js` devServer (non-`/api` only → no duplicate headers on proxied API):

| Header | Value |
|--------|-------|
| `X-Content-Type-Options` | `nosniff` |
| `X-Frame-Options` | `SAMEORIGIN` |
| `Referrer-Policy` | `strict-origin-when-cross-origin` |
| `X-Permitted-Cross-Domain-Policies` | `none` |
| `Permissions-Policy` | `camera=(), microphone=(), geolocation=(), usb=(), payment=()` |

### 2. CSP — Report-Only (HTML only)
- Header `Content-Security-Policy-Report-Only` on HTML documents (backend HTML +
  SPA via devServer). It **reports** but never blocks → zero risk to Stripe /
  PDF preview / File Manager / extension.
- Allowlists Stripe (`js.stripe.com`, `*.stripe.com`, `checkout.stripe.com`,
  `hooks.stripe.com`) and Google sign-in (`accounts.google.com`, `apis.google.com`).
- `style-src 'unsafe-inline'` (Tailwind/React); scripts intentionally NOT given
  `unsafe-inline` so reports reveal exactly what must be nonced for a strict CSP.

### 3. CSP violation sink
- `POST /api/security/csp-report` (public, whitelisted in `access_gate.py`).
- Stored in Mongo `security_csp_reports` with **TTL 30 days** (`ts` index).
- Captures `report` body + client IP (XFF-aware) + UA. Returns `204`.

### 4. Cache-Control
- Private (non-public tier per `classify_path`) `/api/*` responses get
  `Cache-Control: no-store` (auth payloads never cached by shared proxies).
  Existing `Cache-Control` headers are never overwritten.

### 5. HSTS — intentionally NOT set
- Deferred until the prod cut-over to `bibicars.org` over HTTPS (approved).

## Deployment caveat (prod)
In this split deployment the SPA HTML is served by the frontend, not the backend.
Preview coverage is provided via the craco devServer middleware. For **production**
static hosting, the same headers must be emitted at the static-serve / ingress
layer (out of scope for app code) — documented as a prod action item alongside HSTS.
