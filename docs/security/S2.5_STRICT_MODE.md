# BIBI Cars — S2.5 STRICT AUTH MODE

> PHASE SECURITY · Wave S2 · Stage 2.5 — verified on preview, ready for prod.

## What strict mode adds
- Startup invariant assertion (`assert_prod_safe`): refuses to boot if JWT secret / EXT secret missing or CORS is wildcard → fail-fast instead of silent insecurity.
- Disables the `AUTH_MODE=disabled` open-everything bypass.
- **Enforces extension HMAC** (`require_extension_hmac`) — unsigned `/api/ext/*` & `/api/vesselfinder/*` worker calls are rejected (401).

## Exact PROD env contract (set at deploy — NOT committed)
> ⚠️ Note: the env var is **`CORS_ORIGINS`** (not `CORS_ALLOWED_ORIGINS`).

```
AUTH_MODE=strict
JWT_SECRET=<keep existing / strong; auto-persisted if unset>
EXT_SHARED_SECRET=<generate fresh per environment: openssl rand -hex 64>
CORS_ORIGINS=https://bibicars.org,https://www.bibicars.org
# add any real admin/cabinet subdomains if they exist; do NOT use "*"
```

## 🔴 HARD PREREQUISITE before enabling EXT_SHARED_SECRET on prod
Enabling `EXT_SHARED_SECRET` makes extension HMAC **mandatory**. The deployed
**browser extension / VesselFinder worker MUST sign requests with the SAME
secret**, otherwise they will break (401) on production.
- Confirm the extension build's signing secret == prod `EXT_SHARED_SECRET`.
- Generate a **fresh** secret for prod (do not reuse the preview one).
